Per vibe coders

Cos'è Issuebeam

Issuebeam è uno skeleton open source che collega agenti AI a GitHub Issues — Cursor, Claude Code, Copilot e altro. Nasce dal problema reale del vibe coding: bug e task discussi in conversazione e poi dimenticati. La soluzione è una CLI Python leggera più istruzioni agente che la eseguono automaticamente.

Niente dipendenze pesanti, niente gh CLI, niente script PowerShell (spesso bloccati dall'antivirus). Solo stdlib, token e backlog ufficiale su GitHub.

➜ issuebeam in sintesi
[CLI] scripts/github_issue.py
[AGENTS] AGENTS.md · CLAUDE.md · Copilot · Cursor
[LABELS] tracker/labels.yml
[ADOPT] scripts/adopt.py → altri repo
> chat → issue tracciata

Origine

Da Qwibo, generalizzato

Issuebeam è un estratto generalizzato del sistema di tracking usato nel progetto Qwibo (trascrizione audio + riassunto LLM). Da quel contesto sono stati rimossi label e template specifici del prodotto; è rimasta l'architettura che funziona con qualsiasi agente AI su Windows, macOS e Linux: CLI Python eseguibile, token configurabile (env, .env, .secrets), import manifest con Legacy ID, istruzioni multi-piattaforma.

Il nome Issuebeam evoca un raggio che collega la chat al tracker — un ponte diretto tra conversazione e backlog ufficiale.
