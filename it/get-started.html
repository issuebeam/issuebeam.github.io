Setup rapido

Come iniziare

Cinque passi per collegare Cursor a GitHub Issues. Funziona sul repo issuebeam o su un progetto adottato con adopt.py.

01

Token GitHub

Crea un Personal Access Token con permesso Issues: read & write sul repository target.

Metodo consigliato: variabile utente Windows GITHUB_TOKEN. Alternativa: file .secrets/github_token (una riga, gitignored).

# Verifica (dopo aver impostato il token)
python -c "import os; print('OK' if os.environ.get('GITHUB_TOKEN') else 'MANCANTE')"
02

Slug repository

Indica quale repo GitHub usare (owner/nome).

echo myorg/my-app> tracker\github_repo

# oppure in .env
GITHUB_REPO=myorg/my-app
03

Adotta in un altro progetto (opzionale)

Copia lo skeleton in un repo esistente con un comando.

python scripts/adopt.py --target ..\my-repo --repo myorg/my-app
04

Label

Applica le label definite in tracker/labels.yml sul repo GitHub.

python scripts/github_issue.py labels --apply
05

Prima issue + Cursor

Prova la CLI, poi apri il progetto in Cursor — la regola agente è già attiva.

python scripts/github_issue.py create "[Bug] Test da setup" --labels bug
python scripts/github_issue.py list
