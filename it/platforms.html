Non solo Cursor

Piattaforme supportate

Issuebeam non è legato a un solo IDE. Il nucleo è sempre lo stesso: CLI Python + istruzioni per l'agente + GitHub Issues. Cambia solo il file che il tuo tool legge automaticamente.

Documentazione completa: MkDocs — Piattaforme

Architettura

1 · Chat

Descrivi bug e task in linguaggio naturale — da Cursor, VS Code, terminale, …

2 · Agente

Legge AGENTS.md o il file specifico della piattaforma ed esegue github_issue.py.

3 · GitHub

Issue ufficiali con label, template web e chiusura via Fixes #N.

Setup per piattaforma

Cursor

  1. Clona issuebeam o esegui adopt.py nel tuo repo.
  2. Configura GITHUB_TOKEN e tracker/github_repo.
  3. Apri la cartella in Cursor — la regola github-issues.mdc è già attiva.

VS Code + Claude Code

  1. Adotta issuebeam — vengono copiati CLAUDE.md e AGENTS.md.
  2. Apri la root del repository in VS Code (non una sottocartella).
  3. Installa Claude Code e chiedi: «Apri issue per…» — deve eseguire lo script Python.

GitHub Copilot (VS Code / Visual Studio)

  1. Adotta issuebeam — include .github/copilot-instructions.md.
  2. Abilita Copilot Chat con permessi agent/terminal.
  3. Chiedi di tracciare un bug usando la CLI del progetto.

Altre piattaforme

Per Windsurf, Cline, Continue, Gemini CLI, Codex CLI e Aider: copia il contenuto di AGENTS.md nel formato richiesto dal tool (vedi tabella sopra). I requisiti sono sempre gli stessi: Python, token, slug repo, permesso shell.
